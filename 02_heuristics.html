What is a Heuristic?

A heuristic is a rule of thumb вҖ” a problem-specific piece of knowledge that helps guide a search towards a solution faster, without guaranteeing the optimal path.

Formal Definition A heuristic function h(n) is an evaluation function that estimates the cost from node n to the goal node. It uses domain knowledge to assess how "promising" a state is.

Without heuristics, we'd have to explore every possible state blindly (BFS/DFS). With heuristics, we can prioritise states that look more likely to lead to the goal вҖ” dramatically reducing search time.

Key Properties of Heuristics

Admissible Heuristic h(n) вүӨ h*(n) for all nodes. The heuristic NEVER overestimates the actual cost to the goal. This is required for A* to guarantee an optimal solution.
Non-admissible Heuristic h(n) > h*(n) for some nodes. Overestimates the cost вҖ” may find a suboptimal path. Can still be useful but loses the optimality guarantee.
Good Heuristic Doesn't require long computation time. Closely approximates the true cost. More accurately informed = fewer nodes expanded.

How Heuristics Are Defined

Heuristics are typically defined as a simplified/relaxed version of the problem. We pretend certain constraints don't apply, which lets us compute a lower bound on actual cost.

Example вҖ” 8-Puzzle relaxation Real puzzle: a tile can only move to an adjacent blank space.
Relaxed version for "tiles out of place": a tile can move to its goal position in a single step.
This relaxation makes h easy to compute, and since it ignores real constraints, it can only UNDERESTIMATE (= admissible).
Difficulty in practice Experience shows it is VERY DIFFICULT to devise good heuristics. Heuristics are fallible вҖ” they sometimes lead the search astray. No heuristic is perfect.

Why Heuristics Matter: Weak vs Strong Search

Search methods that use heuristics are called "weak" search methods вҖ” not because they're bad, but because the knowledge they incorporate is imprecise. It usually helps but doesn't always.

Without heuristics (BFS/DFS)
  • Explores blindly
  • Complete state space may be needed
  • Very slow for large spaces
  • Guaranteed to find goal (BFS)
With heuristics (Best-First, A*)
  • Guided exploration
  • Dramatically fewer nodes expanded
  • Much faster in practice
  • May miss optimal path (unless admissible)
