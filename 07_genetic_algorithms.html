Genetic Algorithms

Evolution-inspired population-based search

What is a Genetic Algorithm?

Genetic Algorithms (GAs) are inspired by Darwin's theory of evolution. A population of candidate solutions evolves over generations through selection, crossover, and mutation — with fitter solutions more likely to survive and reproduce.

Key Difference from Previous Methods All previous methods (DFS, BFS, SA, Tabu, etc.) are SINGLE-POINT searches — they work with one solution at a time. GAs are MULTI-POINT searches — they maintain an entire POPULATION of solutions simultaneously.

The GA is classified as a metaheuristic and evolutionary algorithm (EA). It's particularly powerful for large, complex search spaces where exact methods are infeasible.

The GA Pipeline

Step 1
Initial Population Generation
Create N random individuals (candidate solutions). Each individual = a chromosome encoding a solution.
↓
Step 2
Fitness Evaluation
Apply fitness function to each individual. Measures how well it solves the problem.
↓
Step 3
Check Termination
Max generations reached? Near-optimal solution found? If yes → return best. If no → continue.
↓
Step 4
Selection
Select individuals to reproduce. Biased toward fitter individuals (tournament selection, fitness-proportionate).
↓
Step 5
Crossover (Recombination)
Selected parents exchange genetic material to produce offspring. Primary operator. High application rate.
↓
Step 6
Mutation
Small random changes to offspring chromosomes. Provides diversity. Low application rate (pm < pc).
↓
Step 7
Population Update
Replace old population with new offspring. Go back to Step 2.

The encoding must be decided BEFORE running the GA. For the Travelling Salesman Problem (TSP), each gene can represent a city — the chromosome encodes the tour order.

Step 2: Fitness Evaluation

The fitness function measures how good a solution is. It's problem-specific and predefined before the run.

What makes a good fitness function?
  • Accurately reflects solution quality
  • Computationally efficient
  • Differentiates between good and bad solutions
  • Guides evolution toward the global optimum
Examples TSP: total tour length (minimize)
Knapsack: total value within weight limit (maximize)
8-queens: number of non-attacking queen pairs (maximize)

Step 4: Selection

Selection is biased toward fitter individuals — but all individuals have some chance of selection to maintain diversity.

Tournament Selection Pick t random individuals from the population. The one with the best fitness wins the tournament and is selected as a parent. Repeat to get the second parent.
Fitness-Proportionate (Roulette Wheel) Selection Each individual's chance of selection is proportional to its fitness. Fitter individuals get a larger "slice" of the roulette wheel.

Step 5: Crossover (Primary Operator)

Two parents exchange portions of their chromosomes to create offspring. Crossover is the primary operator — it's applied at a high rate (pc typically 0.6–0.9).

Single-Point Crossover

Parent 1: │ 8 │ 0 │ 6 │ 2 │ 1 │ 0 │ 0 │ 1 │ ↑ crossover point (after position 4) Parent 2: │ 3 │ 6 │ 1 │ 0 │ 4 │ 9 │10 │ 8 │ Offspring 1: │ 8 │ 0 │ 6 │ 2 │ 4 │ 9 │10 │ 8 │ ← parent1[1:4] + parent2[5:8] Offspring 2: │ 3 │ 6 │ 1 │ 0 │ 1 │ 0 │ 0 │ 1 │ ← parent2[1:4] + parent1[5:8]

A random number r ∈ [0,1] is generated. If r < pc, crossover occurs. Otherwise, parents are copied unchanged.

Crossover is the PRIMARY operator for GAs. It explores the search space by combining good building blocks from two parents. The crossover probability pc is set HIGH (typically 0.6–0.9) because crossover drives convergence.

Step 6: Mutation (Secondary Operator)

Mutation makes small random changes to a chromosome. It's the secondary operator — applied at a LOW rate (pm typically 0.001–0.01).

Why is pm kept low? If pm is too high, the search becomes purely random — destroying good solutions that were built by crossover. Mutation is for fine-tuning and maintaining diversity, not driving the main search.

Flip Bit Mutation (for binary encoding)

Parent: │ 1 │ 0 │ 1 │ 0 │ 0 │ 0 │ 0 │ 1 │ ↑ ↑ ← two bits flipped Offspring: │ 1 │ 0 │ 0 │ 0 │ 1 │ 0 │ 0 │ 1 │

Mutation prevents premature convergence to a local optimum by introducing new genetic material not present in the current population.

Step 7: Population Replacement

Generational Replacement The ENTIRE old population is replaced by new offspring each generation. Clean break between generations.
Steady-State Replacement Only n individuals are replaced each generation. Rest of population stays. Allows faster evolution without waiting for full generation.

Termination

  • Maximum number of generations reached
  • Near-optimal solution quality achieved
  • No significant improvement over recent generations
GAs have NO guarantee of optimality GAs are stochastic — they may not converge, or may converge to a local rather than global optimum. The complete execution (initialisation → specified generations → output) is called a RUN.
