Problem Representation
Trees & Graphs

How AI problems are modelled as state spaces

What is a State Space?

Before any AI search can happen, the problem must be represented as a state space. A state space is a formal model of all possible situations (states) in a problem, connected by legal moves.

State Space = 4 things A set of nodes (each representing a state of the problem) + arcs between nodes (legal transitions) + an initial state (where we start) + a goal state (what we want to reach).

The state space manifests either as a tree or a graph depending on the problem. Search algorithms are then applied to this structure to find a path from initial вҶ’ goal.

Graph vs Tree Representation

Graph

  • States CAN be revisited
  • More than one path between nodes
  • Moves can be undone (bidirectional)
  • Multiple paths to a goal exist
  • Must handle loops & cycles
  • Better for problems like Towers of Hanoi

Tree

  • No revisiting of states
  • Exactly one path between nodes
  • Goal state may appear more than once
  • More memory efficient for some problems
  • No loop detection needed
  • Better for problems like 8-Puzzle

In a TREE, the same goal state may appear more than once since there's only one path per node. In a GRAPH, there's only one instance of each state but multiple paths to reach it.

Classic Problems & Their Representations

8-Puzzle Problem

A 3Г—3 grid with 8 tiles and 1 blank space. The goal is to slide tiles into a target configuration.

Key representation decisions Legal moves are defined as moving the blank space (up/down/left/right), NOT moving the tiles. This keeps the operator count minimal (4 operators) which reduces the branching factor. Fewer operators = smaller state space.

Towers of Hanoi

Move N rings from peg 1 to peg 3. Only one ring at a time, never a larger ring on a smaller one.

State representation Each state = the configuration of all rings on all 3 pegs. Legal moves = moving the top ring from any peg to another, provided no ring violation occurs. Best represented as a GRAPH since moves can be undone.

Travelling Salesman Problem (TSP)

Find the shortest route visiting each city exactly once and returning to start.

Why it's hard With N cities, there are (N-1)!/2 possible routes. This grows factorially вҖ” even 20 cities = ~60 trillion routes. This is why heuristic and metaheuristic methods are needed.

What to Define When Building a State Space

Goal What does the goal state look like? Is it a specific configuration or any state satisfying certain conditions?
Legal Moves / Operators What transitions are allowed? Keep operators minimal вҖ” more operators = bigger branching factor = harder search.
State Description What knowledge must be encoded in each state? Enough to determine all legal next moves.
Best vs Good Enough Some problems (TSP) need the optimal solution. Others (theorem proving) just need any valid solution.

Problem Classification

  • TYPE 1 Problems needing only a representation (e.g. crossword puzzles)
  • TYPE 2 Problems needing a yes/no answer вҖ” does a solution exist?
  • TYPE 3 Problems needing the solution PATH вҖ” goal is known, find how to get there (Towers of Hanoi, theorems)
The ENTIRE tree/graph is NOT pre-generated. Nodes are expanded on-the-fly during search. This is why we need algorithms вҖ” the space is too large to store all at once.
