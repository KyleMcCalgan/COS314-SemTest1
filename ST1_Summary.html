Theory 1 — Problem Representation

AI problems are modelled as a state space: a graph/tree of all possible states reachable from a start state via operators.

State Space = 4 things
  • Initial state
  • Operators / successor function
  • Goal test
  • Path cost
Graph vs Tree
  • Graph — nodes can be revisited (use CLOSED list)
  • Tree — no revisits, simpler but can loop
Problem classification: single-state (fully observable), multi-state (partial), contingency (non-deterministic), exploration (unknown environment).

Theory 2 — Heuristics

A heuristic h(n) estimates the cost from node n to the goal. It guides informed search — the better the heuristic, the less you explore.

Key Properties
  • Admissible — never overestimates (h(n) ≤ true cost). Required for A* optimality.
  • Consistent / Monotone — h(n) ≤ cost(n→n') + h(n'). Implies admissible.
  • h = 0 at goal
8-Puzzle Heuristics
  • h1 — number of misplaced tiles
  • h2 — sum of Manhattan distances
  • h2 dominates h1 (always ≥ h1), so prefer h2
Weak heuristic (h≈0) → lots of search. Strong heuristic → near-direct path. Inadmissible heuristic → fast but not guaranteed optimal.

Algorithms — Uninformed Search

No heuristic. Explore blindly using structure of OPEN list only.

Algorithm OPEN structure Complete? Optimal? Time Space
DFS Stack — children added to LEFT No (may loop) No O(bm) O(bm)
BFS Queue — children added to RIGHT Yes Yes (unit cost) O(bd) O(bd)
DFID DFS with increasing depth limit Yes Yes (unit cost) O(bd) O(bd)
DFS vs BFS: literally one line difference — LEFT (DFS) vs RIGHT (BFS). DFID = best of both: BFS-complete + DFS-space.

Algorithms — Informed Search

Hill Climbing

Idea Always move to the best neighbouring state. No backtracking, no OPEN list history.
  • Uses only h(n) — greedy local move
  • Fast but gets stuck in local maxima, plateaus, ridges
  • Not complete, not optimal

Best First Search (Greedy)

  • Expands node with smallest h(n) — pure heuristic, ignores path cost
  • Not optimal — can find long paths

A* Algorithm

f(n) = g(n) + h(n)
  • g(n) = actual cost from start to n
  • h(n) = estimated cost from n to goal
  • Optimal & complete if h is admissible
  • Expands nodes in order of f(n) — best first on f, not just h
A = Best First + path cost. A* = A with admissible heuristic. If h=0, A* degenerates to uniform cost search (Dijkstra).

Algorithms — Adversarial Search (Games)

Minimax

Idea MAX player picks the move with the highest value. MIN player picks the move with the lowest value. Alternate at each level.
  • Assumes opponent plays optimally
  • Complete (finite game tree), optimal vs optimal opponent
  • Time: O(bm) — explores entire tree

Alpha-Beta Pruning

Prune when β ≤ α
α = best MAX has found so far  ·  β = best MIN has found so far
  • Same result as Minimax — just skips provably irrelevant branches
  • Best case: O(bm/2) — effectively doubles search depth
  • Prune at MIN node: if node value ≤ α → cut.   Prune at MAX node: if node value ≥ β → cut.
Alpha-Beta never changes the Minimax result — it only prunes. The more ordered the moves, the more pruning occurs.

Algorithms — Local Search & Metaheuristics

Used when the path doesn't matter, only the final solution quality. Work by iteratively improving a single (or small set of) solution(s).

Simulated Annealing (SA)

Idea Like Hill Climbing but sometimes accepts worse moves — probability of acceptance decreases as temperature T cools.
  • Accept worse move with probability e−ΔE/T
  • High T → exploratory. Low T → exploitative (like Hill Climbing)
  • Can escape local optima. Theoretically finds global optimum given slow enough cooling

Tabu Search

Idea Hill Climbing + a tabu list that bans recently visited moves/states to force exploration.
  • Always moves to best non-tabu neighbour (even if worse)
  • Tabu list has fixed size — oldest entries expire
  • Aspiration criterion: override tabu if the move produces a new global best

Iterated Local Search (ILS)

Idea Run local search → perturb the solution → run local search again. Repeat.
  • Perturbation must be strong enough to escape the current basin but not so strong it randomises
  • Acceptance criterion decides whether to keep new solution or revert

Algorithms — Genetic Algorithms (GA)

Population-based search inspired by evolution. Maintains a population of candidate solutions (chromosomes) and evolves them over generations.

01
Encoding
Represent solutions as chromosomes (binary, integer, permutation...)
↓
02
Fitness Evaluation
Score each chromosome — higher fitness = better solution
↓
03
Selection
Favour fitter individuals — roulette wheel, tournament, rank-based
↓
04
Crossover
Combine two parents to produce offspring (single-point, two-point, uniform)
↓
05
Mutation
Small random change — maintains diversity, prevents premature convergence
↓
06
Replacement → Repeat
New generation replaces old. Repeat until termination condition met.
Crossover is the primary search operator. Mutation is secondary — too much mutation = random search. Too little = premature convergence.
