Informed search: using heuristics to find better paths

Hill Climbing

Hill climbing is similar to Best-First Search, but only considers local states. Instead of keeping the entire OPEN list sorted globally, it only looks at the children of the current node and picks the best one.

Key distinction from Best-First Best-First: maintains a global OPEN list sorted by heuristic — picks the best node from ALL explored-but-not-expanded nodes.
Hill-Climbing: only reorders the CHILDREN of the current node, places them at the front of OPEN — local focus only.
def hill_climbing(in Start, out State) open = [Start] close = [] State = failure while (open ≠ []) AND (State ≠ success) X ← remove LEFTMOST node from open if X == goal State = success else children ← generate children of X for each child: if not on open or closed: assign heuristic value if on open with shorter path: update if on closed with shorter path: move back to open put X on closed reorder CHILDREN by heuristic (best first, leftmost) place reordered children on LEFT of open // ← LOCAL focus

The Local Optima Problem

Hill climbing's biggest weakness: it can get stuck at a local maximum — a state that's better than all its neighbours, but not the globally best solution.

Three Problems with Hill Climbing 1. Local Optima: Gets stuck in a peak that's not the global best
2. Plateau: Flat area where all neighbours have same value — no direction
3. Ridge: Series of local optima that mislead the algorithm

Greedy Hill Climbing

A variant that always picks the immediate best neighbour without backtracking. Steps:

  • Evaluate the initial state
  • Select a new operator (move)
  • Evaluate the new state
  • If it's closer to goal → make it current state
  • If not better → ignore it
  • If at goal or no operators left → quit
  • Otherwise repeat from step 2
Greedy hill climbing WITHOUT backtracking can miss the goal entirely if there's a local max in the way. WITH backtracking, it keeps a list of promising nodes to revisit. This avoids getting permanently stuck.

The A Algorithm

The A algorithm is Best-First Search using a specific evaluation function that combines both the cost to reach the current node AND the estimated cost to the goal.

f(n) = g(n) + h(n)
g(n) = actual cost from start to node n (path length so far)
h(n) = heuristic estimate from n to the goal

g(n) The actual, known cost of getting from the START node to node n. Usually the depth in the tree, or sum of arc costs. Tells us how expensive the path so far is.
h(n) The heuristic estimate of cost from n to the GOAL. Uses domain knowledge. For 8-puzzle: number of tiles out of place.

The A* Algorithm

A* is the A algorithm where the heuristic h(n) is admissible: it never overestimates the actual cost. This guarantees finding the optimal (shortest) path.

f*(n) = g*(n) + h*(n)
g*(n) = true shortest cost from start to n
h*(n) = true cost from n to goal (unknown in practice)

We estimate: g(n) ≤ g*(n) and h(n) ≤ h*(n)
When is the A algorithm called A*? When h(n) ≤ h*(n) for ALL nodes — i.e., the heuristic is ADMISSIBLE (never overestimates). This single condition is what makes A* guaranteed to find the optimal solution.

A* on the 8-Puzzle — worked example

Using h(n) = tiles out of place, g(n) = depth in search tree State a (initial): f(a) = g(0) + h(5) = 0 + 5 = 5 ┌───┬───┬───┐ │ 2 │ 8 │ 3 │ ← g=0 (at start) │ 1 │ 6 │ 4 │ ← h=5 (tiles 2,8,1,6,blank not in place? Actually count misplaced) │ 7 │ │ 5 │ └───┴───┴───┘ State b (1 move): f(b) = g(1) + h(5) = 1 + 5 = 6 State c (1 move): f(c) = g(1) + h(3) = 1 + 3 = 4 ← BEST, expand this next State d (1 move): f(d) = g(1) + h(5) = 1 + 5 = 6 A* always expands the node with LOWEST f(n) value from OPEN list

Admissibility of 8-Puzzle Heuristics

Both "tiles out of place" and "Manhattan distance" are admissible for the 8-puzzle because they count moves assuming tiles can pass through each other — a relaxed problem that always gives a lower bound.

h(n) = tiles out of place is admissible because each misplaced tile needs AT LEAST 1 move to reach its goal — we never overestimate. Manhattan distance is even better (more informed) but also admissible.

A* Advantages and Disadvantages

AdvantagesDisadvantages
✅ Optimal: Guarantees shortest path if h is admissible ❌ Memory: Stores all explored nodes in OPEN/CLOSED
✅ Complete: Will find a solution if one exists ❌ Heuristic dependent: Bad h(n) → bad performance
✅ Efficient: Explores most promising paths first ❌ Complex: Careful data structure management needed
✅ Widely applicable: Navigation, game AI, robotics, networking ❌ If h overestimates: may find suboptimal path
