Minimax Algorithm
& Alpha-Beta Pruning

Game tree search for two-player zero-sum games

Minimax Algorithm

Minimax is used in two-player zero-sum games (chess, tic-tac-toe, Go) where one player winning means the other loses. Both players play optimally.

MAX Player Wants to MAXIMISE their score. Picks the child with the HIGHEST value. Assumes worst-case: starts at -∞.
MIN Player Wants to MINIMISE MAX's score. Picks the child with the LOWEST value. Assumes worst-case: starts at +∞.

The Algorithm

Minimax uses DFS to traverse the entire game tree, assigning values upward:

MINIMAX ALGORITHM: Repeat: 1. If at a leaf node (or depth limit reached): return the static value of the state 2. If at a MIN level: apply minimax to all children return the MINIMUM of the results 3. If at a MAX level: apply minimax to all children return the MAXIMUM of the results Until entire tree is traversed

Worked Example — Step by Step

Tree structure: A ← MAX level / \ B C ← MIN level / \ / \ D E F G ← MAX level /\ /\ /\ /\ 3 8 9 2 4 -8 8 2 ← leaf values (utility) Step 1: DFS to leaves. Propagate up. D = max(3, 8) = 8 (MAX node picks highest) E = max(9, 2) = 9 (MAX node picks highest) F = max(4, -8) = 4 (MAX node picks highest) G = max(8, 2) = 8 (MAX node picks highest) Step 2: MIN level B = min(D=8, E=9) = min(8, 9) = 8 C = min(F=4, G=8) = min(4, 8) = 4 Step 3: MAX level (root) A = max(B=8, C=4) = max(8, 4) = 8 RESULT: MAX should choose to go to B (value = 8) Note: Even though E=9 exists under B, MIN prevents MAX from accessing it.
MAX always picks the child with HIGHEST value. MIN always picks LOWEST. The values bubble up from leaves. The root value tells you the optimal outcome assuming both play perfectly.

Characteristics of Minimax

  • Uses DFS internally to traverse the tree
  • Always finds optimal path in a finite game tree
  • Struggles with high branching factor (chess has ~35 moves per turn)
  • For large games, trees are generated only to a fixed depth (ply)
  • At cut-off depth: use a heuristic value instead of win/draw/loss
  • Horizon effect: a promising path that's not detected within the search depth
Leaf values in full vs limited search FULL search: 1 (win), 0 (draw), -1 (loss) assigned to terminal states.
DEPTH-LIMITED: use a heuristic function at the cutoff depth. In chess: count of remaining white vs black pieces (positive = good for MAX, negative = bad).

Alpha-Beta Pruning

Alpha-beta pruning optimises Minimax by skipping branches that cannot affect the final decision. It returns the SAME result as Minimax but explores far fewer nodes.

IF α ≥ β → PRUNE the branch
Stop exploring this subtree — it won't change the result
Alpha (α) Best value that MAX can guarantee. Starts at -∞. Only updated by MAX nodes (going up). MAX will NEVER accept a value less than α.
Beta (β) Best value that MIN can guarantee. Starts at +∞. Only updated by MIN nodes (going up). MIN will NEVER accept a value greater than β.

Pruning Logic

  • Beta cutoff (α-cut): At a MAX node, if α ≥ β, prune remaining children. MIN already has a better option elsewhere — MAX will never get to use this branch.
  • Alpha cutoff (β-cut): At a MIN node, if β ≤ α, prune remaining children. MAX already has a better option elsewhere — MIN will never allow this path.

Alpha-Beta Algorithm

procedure MINIMAX(node, α, β): if node is a leaf: return value(node) if node is MAX: for each child c: α ← max(α, MINIMAX(c, α, β)) if α ≥ β: BREAK // Beta cutoff — prune remaining children return α else (node is MIN): for each child c: β ← min(β, MINIMAX(c, α, β)) if β ≤ α: BREAK // Alpha cutoff — prune remaining children return β

Step-by-Step Trace Example

Tree: A(MAX) → B(MIN) → D(MAX) → H(MIN)[4,5], I(MIN)[3,7] α=-∞, β=+∞ Step 1: DFS to H (MIN node). β = min(+∞, 4) = 4. Check next child: 5. β = min(4, 5) = 4. Node H = 4. Step 2: Backtrack to D (MAX node). α = max(-∞, 4) = 4. Proceed to I. Pass α=4, β=+∞ down. Step 3: At I (MIN node). β = min(+∞, 3) = 3. Now check: α=4 ≥ β=3 → PRUNE! Don't explore I's second child. I = 3. Step 4: Back at D: α = max(4, 3) = 4. D = 4. Step 5: Back at B: β = min(+∞, 4) = 4. B provisionally = 4. (Continue to E branch... see that E=9 but min(4,9)=4, so B=4 confirmed) Step 6: Back at A: α = max(-∞, 4) = 4. Explore C branch with α=4, β=+∞... PRUNING SAVED US: We didn't need to evaluate I's second child! Same result as full minimax, fewer nodes explored.
Alpha-Beta returns IDENTICAL results to Minimax — it just skips provably irrelevant branches. Best case: it can reduce nodes from O(b^d) to O(b^(d/2)), effectively doubling the searchable depth!

Key Rules to Remember

Pass α and β down the tree Every recursive call inherits the current α and β values from its parent. α can only increase (set by MAX nodes). β can only decrease (set by MIN nodes). When α ≥ β at any node → prune that branch.
α inherited from below When backtracking up, a MAX node's α is set to the max of its children. A MIN node's β is set to the min of its children. These values propagate upward.

Comparison: Minimax vs Alpha-Beta

Minimax
  • Explores EVERY node to cutoff depth
  • Simple to implement
  • Correct but slow
  • O(b^d) nodes
Alpha-Beta
  • Skips irrelevant branches
  • Slightly more complex
  • Same result, much faster
  • Best case O(b^(d/2))
